## Project Overview

DesiCalc — a collection of **free Indian tax and financial calculators** (Income Tax, Stamp Duty, RTO Road Tax) hosted at [desicalc.in](https://desicalc.in).

The entire site is **static HTML** with zero framework overhead. All calculations run client-side in vanilla JavaScript. No build step exists beyond Tailwind CSS compilation.

## Build & Dev Commands

```bash
# Install dependencies (first time only — just Tailwind CSS CLI)
npm ci

# Build Tailwind CSS for production (minified, purged)
npm run build

# Watch Tailwind for changes during development
npm run dev

# Regenerate MCP/A2A agent discovery files (after editing api-config.js)
npm run generate-discovery

# Serve the public/ directory locally
npx serve public
```

### Key Architectural Patterns

1. **Fully client-side**: All calculations happen in the browser. No server round-trips. Each tool page is self-contained HTML that loads calc-core.js + its domain logic.

2. **Progressive enhancement**: Forms work without JS (static display), live calculation enhances when JS loads.

3. **SEO pages are hand-authored HTML**: Each state-specific page under `stamp-duty/` and `rto-tax/` is a standalone HTML file with full structured data, FAQ schema, breadcrumbs, and canonical/hreflang links. No static site generator — these are written directly.

4. **i18n via data attributes**: Elements use `data-i18n="key"` attributes. The `common.js` script toggles `data-lang` on `<html>` and replaces `innerText` by looking up keys in `window.DESICALC_LANG`. Preference persists to `localStorage`.

5. **State management per tool**:
   - URL search params — input sharing/bookmarking
   - localStorage — restore last inputs on return
   - sessionStorage — ad dismiss state per tab session

6. **Cloudflare Functions middleware** (`functions/_middleware.js`): Intercepts requests with `Accept: text/markdown` header and converts HTML pages to Markdown on-the-fly for AI/agent consumption. Also serves MCP endpoints under `/mcp`.

7. **PWA caching strategy**: Cache-first for static assets (CSS, JS, images), network-first for HTML pages. Offline calculator use works after first visit.

### How Each Calculator Works

- **Income Tax**: Reads income/regime/deduction inputs → applies tax slabs from `slabs-2026.js` (new regime: 7 slabs up to 30%; old regime: 3-4 slabs depending on age) → applies 87A rebate → applies surcharge (based on income bracket) → adds 4% health & education cess.
- **Stamp Duty**: Reads state/property value/gender → looks up `STAMP_DUTY_RATES` in `state-rates.js` (28 states, varies by gender/joint ownership) → adds registration fee, metro cess, LBC, surcharge where applicable.
- **RTO Tax**: Reads state/vehicle type/price/fuel → applies state-specific rate structure (slab %, flat %, or per-CC) → adds registration fee + HSRP. EVs get exemption per state policy. Commercial vehicles use seating-based brackets.

### SEO Patterns

Each HTML page follows a consistent SEO template:
- Canonical URL + hreflang links (en, hi, x-default)
- Open Graph tags (title, description, image, type)
- Twitter card
- JSON-LD structured data (WebApplication/FAQPage/BreadcrumbList/WebSite/ItemList depending on page type)
- Semantic HTML5 (`<main>`, `<nav>`, `<article>`, `<section>`)

### MCP/A2A Agent Integration

- Central API registry in `api-config.js`
- Run `npm run generate-discovery` after adding/updating APIs to regenerate:
  - `.well-known/api-catalog` (linkset format)
  - `.well-known/agent-card.json` (Google A2A format)
  - `.well-known/mcp/catalog.json`
  - `.well-known/mcp/server-card.json`
  - `public/mcp/server-card` and `public/mcp/server-card.json`
